Lesson 2

Creating a Structured Document with HTML
The input to webpage development is a webpage design as PDF/image file. The output is the result of Frontend development which involves HTML, CSS, JavaScript. In this nanodegree we only cover HTML and CSS.
The browser uses HTML document to build a tree-like structure called DOM (Document Object Model), each node of which is an HTML element. Every HTML element in a webpage is rendered in a rectangular area of the webpage.

Languages:
HTML: language for the "structure" of a webpage.
CSS: language for the "style" (appearance) of a webpage.

Tools:
Besides the languages (HTML and CSS), Tools are very important for webpage development.
Editing tool:
--scratchpad.io is a good editor to start development since it is easy and feedback is immediate. It can not be used to share the web page though.
--codepen.io is an intermediate level editing tool. It allows sharing and saving of developed webpage. We need to create an account on its web page though.
--Sublime Text 2 is the tool of choice for Professional webpage development.

Troubleshooting/Debugging tool:
Chrome has very powerful "Developer Tool" that when launched from Menu appears as structure and style panels in addition to the panel where the webpage is rendered. The panels allow in-place changing of structure and style to see the effect on the resulting webpage.
Developer Tool is actually a Toolbox. One of the tools in the toolbox allows inspection of structures and styles of various elements of the webpage. In the reverse direction hovering over a structural element in HTML panel highlights the box in the result panel.

Development steps:
Everything in a webpage is a box. Start development from one big box that is the entire webpage, and progressively specify details and positions of smaller boxes; each box should be defined with a "div" tag. The contents of boxes are filled next. To add style give each "div" tag a "class" attribute with informative name.

Lesson 3

General:
-CSS (Cascading Style Sheets)code is used to add style (appearance) to HTML elements on a webpage; "Cascading" means for each HTML element the more specific style rule ultimately applies
CSS rules:
<selector> determines the elements to which the rules apply such as
"element tag":
"class (ex. .description)": used for styling "many similar elements of possibly different tags". The use of class helps ""avoid repetition" of CSS code.
"id (#lesson1)": used to style a "unique" element; although possible to use it like a class it is not recommended. Uniqueness is also used by JavaScript.

-each rule is a line of the form property:value
<selector> {
property: value;
...
}

Some useful rules:
-"p {line-height: ...}" - controls space between consecutive lines of text
-"background-image: url(images/myImage.png)"

-various CSS properties are listed at: http://developer.mozilla.org/en-US/docs/Web/CSS/Reference
Inheritence:
an element "inherits" style rules from its parents which it can override; Browsers have default styles builtin which can be overridden.
-descendent elements "inherit text-relatd properties" such as: color, font, letter-spacing, line-height, list-style, txt-align, text-indent, text-transform, visibility, white-space, word-spacing
-descendent elements "do not inherit box-related properties": background, border, display, float and clear, height and width, margin, min/max height/width, outline, overflow, padding, position, text-decoration, vertical-align, z-index
The Box Model:
-A "box" is basis of applying CSS rules. Boxes have 4 components (from outermost):
Margin: to provide space between boxes; it is transparent (no background color)
Border: our size of element box with rule "box-sizing: border-box"
Padding: clears an area around the content
Content:

-div and span don't have any "default" style applied by the browsers
-"div" elements are for layout control and group things together; their default width is browser width; with "width: ##%" the box resizes as browser is resized, if specified as number of pixels (##px) then no box resizing happens; "max-width" is used to specify an upward bound on a resizeable box size.
-"span" elements are for decorating (styling) elements inline
div { display: flex; } - this rule allows child elements to be placed side by side if space permits; by default "div" elements occupy entire width of the browser
Development Process:
-First structure a webpage; look for natural boxes from Design document (a pdf or a graphic file), then needed HTML tags, followed by sizing and positioning the boxes; the result is a layout
-look for repeated styles and semantic elements (choose the right tag based on its purpose on the webpage)
-write HTML
-apply styles (from biggest to smallest boxes)
-Fix issues
-all resources of a webpge should be at or below the root folder of webpage/site
-Developer Tools helps in refinement by allowing experimentation with immediate
visual feedback
-"* { outline: 1px solid red}" CSS rule puts boxes around elements which can be a visual feedback during development
